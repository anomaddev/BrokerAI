# BrokerAI v0.0.1 ‚Äî Alpha

> **Historical note:** Point-in-time alpha notes. Current stack uses Postgres/Supabase and persistent bots `secretary`, `broker`, `researcher`. See [`docs/README.md`](../README.md).

**Release date:** 2026-06-23  
**Tag:** `v0.0.1`  
**Commit:** `ff6b9ab`

First public alpha of BrokerAI ‚Äî a multi-bot trading platform scaffold for Proxmox LXC, with an auth-gated web dashboard, CLI, local MongoDB cache, and GitHub-based auto-updates.

> **Note:** Alpha 0.0.1 is a structural foundation. Trading bots and sub-brokers are stubs; production trading is not supported in this release.

---

## Highlights

- **One-command Proxmox install** via `ct/brokerai.sh` (LXC + full stack)
- **Standalone container install** via `scripts/install-lxc.sh`
- **Web dashboard** on port **1989** with setup wizard, login, and settings UI
- **`brokerai` CLI** for status, bot control, updates, and services
- **Orchestrator + IPC** for sub-bot lifecycle and heartbeat
- **MongoDB** local cache with indexed collections
- **Auto-update** systemd timer with Swift Package Manager-style update tracks

## Installation

### Proxmox (recommended)

Run on the Proxmox host:

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/anomaddev/BrokerAI/main/ct/brokerai.sh)"
```

Optional pre-set admin credentials:

```bash
BROKERAI_ADMIN_USER=admin BROKERAI_ADMIN_PASSWORD='YourStr0ng!Pass' \
  bash -c "$(curl -fsSL https://raw.githubusercontent.com/anomaddev/BrokerAI/main/ct/brokerai.sh)"
```

### Standalone (Debian/Ubuntu container or VM)

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/anomaddev/BrokerAI/main/scripts/install-lxc.sh)"
```

After install, open `http://<host>:1989` and complete setup (or sign in with pre-provisioned credentials).

## CLI

```bash
brokerai help
brokerai status
brokerai bots list
brokerai bots start|stop <name>
brokerai update check
brokerai services restart   # root
brokerai version
```

JSON output: add `--json` to `status`, `bots`, and `version`.

## Update tracks

Configure in **Settings ‚Üí System ‚Üí Updates** or `/etc/brokerai/config.env`:

| Track | Description |
|-------|-------------|
| `branch` | Latest commit on `BROKERAI_BRANCH` (default: `main`) |
| `release` | Pinned to a specific tag (e.g. `0.0.1`) |
| `latest-release` | Newest GitHub release tag |

Auto-update runs on a systemd timer (every 6 hours). Manual check/apply via UI or `brokerai update check` / `scripts/update-now.sh`.

## Architecture

| Component | Role |
|-----------|------|
| **Orchestrator** | Sub-bot lifecycle, heartbeat, control IPC |
| **Web API** | FastAPI + React SPA |
| **MongoDB** | Local cache (`market_data`, `research_cache`, `analysis_results`) |
| **Sub-bots** | `brokers`, `researcher`, `data_manager`, `data_analyzer`, `executor` |

**Brokers sub-brokers (stubs):** crypto, forex, stocks, futures, options

## Known limitations (Alpha)

- All sub-bots and sub-brokers are **stubs** ‚Äî no live trading
- Settings tabs beyond **System** are placeholders
- Requires MongoDB (installed automatically on container deploy)
- `latest-release` track requires at least one GitHub release or tag on this repo
