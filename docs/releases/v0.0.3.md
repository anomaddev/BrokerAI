# BrokerAI v0.0.3 — Alpha

> **Historical note:** Point-in-time alpha notes. Current stack uses Postgres/Supabase and persistent bots `secretary`, `broker`, `researcher`. See [`docs/README.md`](../README.md).

**Release date:** 2026-06-25  
**Tag:** `v0.0.3`  
**Commit:** `0628246`

Alpha 0.0.3 turns BrokerAI from a scaffold into a research-aware trading platform: multi-model daily forex reports, a live exchange dashboard, OANDA connectivity, and a fully wired settings UI for models, data sources, and broker asset classes.

> **Note:** Alpha 0.0.3 adds real research and exchange integration, but **does not execute live trades**. Only forex daily research is fully implemented; other asset classes remain stubs.

- **Daily research reports** — multi-model contributor analysis, synthesis, per-model outputs, and rolling weekly briefs

## Research

### Daily report pipeline

The researcher bot runs a scheduled daily workflow (configurable in **Settings → Daily Reports**):

1. **Contributor models** — each enabled model independently analyzes enabled forex pairs, grouped by primary currency
2. **Synthesis model** — merges contributor outputs into a single institutional daily brief
3. **Per-model reports** — individual markdown files saved alongside the synthesized report
4. **Weekly brief** — appends each day's synthesis into a rolling weekly document

Data sources (toggle per capability in **Settings → Connections**):

| Source | Description |
|--------|-------------|
| **NewsAPI** | Financial news articles with progressive query broadening per currency group |
| **Web search** | Model-driven web search (requires a model with the capability enabled) |
| **X search** | Social sentiment via model capability (treated as corroboration, not primary news) |

### Scheduling

Reports run relative to a market open time, not a fixed UTC hour:

- Choose a reference market: London, New York, Tokyo, Sydney, Frankfurt, Hong Kong, Singapore
- Set an offset from open (default: **London −2 hours**)
- Enable/disable scheduling with a toggle; the researcher bot checks once per tick after the scheduled window

Weekend runs produce Monday-open positioning guidance rather than intraday execution notes.

### Reasoning effort

Contributor and synthesis models support configurable reasoning depth: **None**, **Low**, **Medium**, or **High** (default: High for synthesis).

### Signals API

`GET /api/research/signals` parses the latest synthesized daily report and returns structured per-pair signals:

- **Signal** — buy, sell, hold, mixed
- **Tone** — bullish, bearish, neutral
- **Approach** — strategy summary
- **Conviction** — low, medium, high

Forex is fully implemented; other asset classes appear in the snapshot with `not_implemented` status.

### Report storage

Reports are written to `data/research/reports/<year>_<week>/`:

| Type | Filename pattern | Description |
|------|------------------|-------------|
| `daily` | `YYYY-MM-DD-daily.md` | Synthesized daily brief |
| `daily_model` | `YYYY-MM-DD-daily_<model-slug>.md` | Per-contributor output |
| `weekly` | `weekly_report_<week>.md` | Rolling weekly synthesis |

## Web UI

### Dashboard

The home page is now an **exchange dashboard**:

- Connected exchange cards with environment badge (Practice / Live)
- Account summary: balance, NAV, unrealized/realized P&L, margin used/available
- Open trade, position, and pending order counts
- Enabled asset classes per exchange with links to broker settings
- Skeleton loading states while data fetches

### Daily Reports (`/daily-reports`)

- **Signals panel** at the top — tabbed by asset class with conviction badges
- **Report library** — search, filter by type (daily, per-model, weekly), sort by date
- **Report viewer** (`/daily-reports/r/*`) — rendered markdown with metadata (model, reasoning effort, generated-at)

Legacy `/research` routes redirect to `/daily-reports`.

### Settings

Navigation reorganized into five sections:

| Section | Tabs |
|---------|------|
| **General** | General (placeholder) |
| **Data** | Models, Connections |
| **Research** | Daily Reports |
| **Broker** | General, Forex, Precious Metals, Stocks, Crypto, Futures, Options |
| **System** | System (bots, MongoDB, updates, power) |

**Models** — add, edit, enable/disable, test, and delete AI model connections. Supported providers:

- Open WebUI (local Ollama / Open WebUI)
- OpenAI
- Claude (Anthropic)
- Grok (xAI)

**Connections** — NewsAPI key management and OANDA exchange setup in one place. Model capability toggles (web search, X search) are configured per model.

**Daily Reports** — contributor model roster, synthesis model, data source toggles, schedule market/offset, enable scheduling.

**Broker asset tabs** — enable/disable each asset class, select instruments (forex pair grid), assign a primary exchange from connected providers.

Settings panels use **auto-save** with debounced persistence and inline save status.

### Sidebar

- Grouped navigation: Dashboard, Research & Analysis, External links, System
- Daily Reports under "Research & Analysis"
- Settings under "System"

## OANDA integration

Connect OANDA via **Settings → Connections**:

- **Practice** or **Live** environment
- Personal access token (stored securely; never returned after save)
- Account picker populated from a live connection test
- Re-test saved credentials or test before saving

API endpoints:

```
GET/PUT/DELETE  /api/settings/exchanges/oanda
POST          /api/settings/exchanges/oanda/test
POST          /api/settings/exchanges/oanda/test-connection
GET           /api/settings/exchanges/oanda/account-summary
```

OANDA supports **Forex** and **Precious Metals** asset classes. Other exchanges (Interactive Brokers, MetaTrader 5, Binance, Coinbase, Kraken) appear in the catalog as coming soon.

## CLI

New `research` command group:

```bash
brokerai research status              # readiness: models, sources, brokers, last run
brokerai research run-daily           # trigger a daily report now
brokerai research run-daily --force   # ignore last-run-date guard
brokerai research list                # list markdown reports
brokerai research show <date|filename>  # print report content
brokerai research test-news           # verify NewsAPI connection
brokerai research test-model          # verify contributor model connectivity
```

Add `--json` to any subcommand for machine-readable output.

## Upgrade from v0.0.2

1. Set your update track to `release` with pin `0.0.3`, or pull `main` if on `branch` track
2. Apply the update via **Settings → System → Updates** or `brokerai update check`
3. Configure at minimum:
   - One AI model in **Settings → Models**
   - NewsAPI key in **Settings → Connections** (if using news)
   - Enabled forex pairs in **Settings → Broker → Forex**
   - Contributor + synthesis models in **Settings → Daily Reports**
4. Optionally connect OANDA in **Settings → Connections** for the exchange dashboard

No database migration script is required — new collections are created on first access.

## Known limitations (Alpha)

- **No live order execution** — research and account visibility only
- **Forex research only** — metals, stocks, crypto, futures, and options are scaffolded but not analyzed
- **OANDA only** — other exchanges are catalog placeholders
- **General settings** tab is still a placeholder (bot name, timezone, etc.)
- **Claude** provider does not support in-app connection testing
- Scheduled reports require the researcher bot to be running (`brokerai bots start researcher`)
