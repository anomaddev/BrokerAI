# BrokerAI v0.0.8 ‚Äî Alpha

**Release date:** 2026-07-06  
**Tag:** `v0.0.8`  
**Commit:** _(set at release)_

Alpha 0.0.8 is a **strategy analysis workspace and market calendar** release. It adds first-class UI for browsing and inspecting strategy analysis runs, recency-aware badges and navigation, rich candle charting with OHLC legends and forex market open/close markers, a reusable market calendar module, the `at_time` indicator, and continued refinements to the EMA crossover preset, trade reason system, and Secretary pipeline.

> **Note:** Live trading remains **forex-only via OANDA**. This release focuses on analysis tooling, chart UX, and supporting library work ‚Äî not on new asset classes or execution paths.

---

## Highlights

- **Analysis run workspace** ‚Äî New routes, detail panel, navigation helpers, and filter multi-select for strategy analysis runs
- **Recency & navigation** ‚Äî `AnalysisRecencyBadge`, run recency utilities, and candle window helpers for focused analysis views
- **Enhanced candle charts** ‚Äî `ExploreCandleChart`, `ChartOhlcLegend`, market closure boundaries (weekly + daily), and time localization updates
- **Market calendar** ‚Äî `marketCalendar.ts` + tests for forex trading sessions, DST-aware session logic, and weekly gap detection
- **`at_time` indicator** ‚Äî New trading indicator for time-of-day conditions
- **Strategy & signal refinements** ‚Äî Updates to EMA crossover definition, filters, signal logic, and trade reason registry
- **Secretary / pipeline** ‚Äî Candle timeline, analysis worker, and execution gate improvements
- **Candle cache & time** ‚Äî `candleTime.ts`, cache updates, and timeframe utilities

## Pull request summary

Use this section as the PR description body.

### What changed

BrokerAI gains a dedicated strategy analysis section in the dashboard with run listing, filtering, recency indicators, and deep-link navigation to individual runs with candle context. Charts now render market session boundaries and OHLC legends. A new market calendar library provides session open/close calculations and test coverage. Supporting changes touch the EMA crossover preset, trade reasons, Secretary pipeline, broker gates, and data analyzer worker.

### Why

- Surface analysis run history and state for operators and researchers
- Improve chart readability with session markers and legends for forex instruments
- Centralize market-hour logic (America/New_York, DST, weekends) for consistent candle handling
- Prepare richer strategy analysis tooling ahead of multi-strategy dashboards

## State of the bots (0.0.8)

**Persistent bots** remain: `secretary`, `broker`, `researcher`.

**Data model:** `broker_lots` + `broker_events` + `broker_sync_state` (from 0.0.7) + new `analysis_runs` collection for persisted strategy analysis.
